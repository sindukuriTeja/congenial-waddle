1. Introduction

1.1 Purpose

PestiVid is a Web3 demo platform for agricultural transparency and traceability. Farmers record/store crop practice videos on IPFS (via Pinata), linked to a simulated Solana blockchain. Buyers browse video-verified produce for simulated purchases; investors fund projects with transparent (simulated) returns. It showcases Web3's potential to enhance trust, efficiency, and direct engagement in agriculture.

  • Structure: Clear, hierarchical headings and numbering.

1.4 Project Scope

PestiVid demo's scope:

  • Authentication: Simulated multi-factor login (Orange ID, Solana wallet).
  • Role-Based Access: Distinct interfaces for Farmers, Buyers, Investors.
  • Video Evidence: Farmer video recording, metadata, IPFS upload (Pinata) with SHA256 hash.
  • Marketplace (AgriSell): Farmers list produce with video; Buyers browse/simulate purchases.
  • Funding Platform: Farmers request project funding; Investors browse/simulate investments.
  • AI Integration: Plant disease recommendation (Gemini API); AgriBot chatbot (Groq API).
  • Simulated Blockchain: Demonstrates transaction recording (Solana simulated).
  • Communication: Secure messaging, notifications.
  • Demo Data Persistence: Browser localStorage simulates backend.

Aims to show enhanced traceability, direct market access, and transparent investment in a simulated Web3 agricultural ecosystem.

2. Overall Description

2.1 Product Perspective

PestiVid is a self-contained demo showcasing Web3 in agriculture. It's a proof-of-concept for transparency, traceability, and direct engagement, not a replacement for existing systems. Its modular features could be adapted for larger platforms.

It addresses trust/efficiency needs in agricultural transactions, demonstrating how video evidence, decentralized storage, and simulated blockchain mitigate information asymmetry and verification/investment barriers.

A real-world version might interface with actual blockchains, identity solutions, payment gateways, data oracles, and supply chain systems.

2.2 Product Features

Key PestiVid demo features:

  1. User Auth & Role Management: Secure login (Orange ID, Solana), role selection, profile management.
  2. Farmer Video & IPFS Upload: Record videos, add metadata, upload to IPFS (Pinata) with SHA256.
  3. Farmer AgriStream: Gallery of uploaded videos; initiate listings/funding.
  4. AgriSell Marketplace: Farmers list produce with video; Buyers browse, simulate purchases.
  5. Farmer Funding Platform: Farmers create funding requests with video.
  6. Investor Project Discovery & Investment: Investors browse requests, view videos, simulate investments.
  7. Investor Portfolio: Track (simulated) investments, progress, history.
  8. AI Plant Disease Recommendation: Farmers upload images for Gemini API analysis.
  9. AI Farming Assistant (AgriBot): Groq API-powered chatbot for farming advice.
  10. Secure Messaging: 1-to-1 text communication.
  11. User Profiles & Activity: View profiles and role-specific activity.
  12. Notifications: Toast notifications for platform events.

2.3 User Classes and Characteristics

  1. Farmers: Agricultural producers; seek to showcase practices, reach buyers/investors, access modern tools. Need intuitive video/listing/funding tools, useful AI.
  2. Buyers: Individuals/businesses seeking produce; value transparency, verification, direct sourcing. Need easy browsing, clear info with video, simple (simulated) purchase.
  3. Investors: Individuals/entities investing in projects; seek transparency, verifiable details, ROI info. Need detailed listings with video, clear terms, (simulated) tracking.

All three user classes are primary. No "less important" classes for this demo.

2.4 Operating Environment

  • Web app for modern browsers (desktop/mobile). Standard hardware.
  • Stable internet for API/media uploads.
  • localStorage for demo data persistence.
  • Solana browser wallet (e.g., Phantom) for Solana interaction.
  • Camera access for video recording.

2.5 Design and Implementation Constraints

  1. Tech Stack: Vue.js 2, Tailwind CSS. Integrates: Orange ID, Solana Web3.js, Pinata, Gemini, Groq APIs (Axios).
  2. Demo Persistence: Data in browser localStorage (not scalable).
  3. Simulated Blockchain: Solana interactions simulated; no real crypto.
  4. API Key Management: CRITICAL: Client-side keys for demo. Production MUST use backend key management.
  5. IPFS Public Nature: Data public by CID.
  6. SPA: Client-side routing.
  7. Browser Compatibility: Tested on modern Chrome.
  8. Demo Security: Not secure for real data/transactions due to client-side nature.

2.6 User Documentation

Demo documentation: Intuitive UI, clear labels, in-line tooltips. This SRS is technical documentation. No separate user manuals.

2.7 Assumptions and Dependencies

  1. API Stability: Continued operation of Orange ID, Pinata, Gemini, Groq.
  2. API Quotas/Billing: Usage within free tiers or dev handles billing. Valid API keys needed.
  3. Internet: Users need stable connection.
  4. Modern Browser: JS enabled, localStorage access.
  5. Solana Wallet: For Solana login features.
  6. Camera Permissions: For video recording.
  7. localStorage Integrity: Assumed not disabled/cleared for data persistence.
  8. Correct API Keys: Developer uses valid keys.

3. System Features

3.1 User Authentication and Role Management

Desc: User auth (Orange ID, Solana Wallet), role management (Farmer, Buyer, Investor), PestiVid profile. Priority: High.

Stimulus/Response:

  • Orange ID Login: User initiates. Redirect to Orange ID. On success, back to PestiVid. Capture details. New users: Solana & role prompt.
  • Solana Wallet Connect: User clicks. Wallet prompts. On approval, capture public key. New users: role prompt.
  • New User Onboarding: Post-auth, new user sees form. Selects role, enters PestiVid name/email. Profile to localStorage. To role dashboard.
  • Logout: User initiates. Session cleared. Orange ID logout if applicable. To landing.

Functional Req:

  • REQ-3.1.1: Support Orange ID auth.
  • REQ-3.1.2: Support Solana wallet connection.
  • REQ-3.1.3: New PestiVid users provide name, email, select role post-auth.
  • REQ-3.1.4: User roles/profile to localStorage (keyed by ID).
  • REQ-3.1.5: Role-based navigation/feature access.
  • REQ-3.1.6: Allow logout, clear session, trigger Orange ID logout.

3.2 Farmer: Video Recording & IPFS Upload (PestiVid Module)

Desc: Farmers record crop/practice videos, add metadata, upload to IPFS (Pinata) with SHA256. Priority: High.

Stimulus/Response:

  • Farmer to "PestiVid," fills details, clicks "Start Recording." -> Camera access. Live feed. Record.
  • Farmer clicks "Stop Recording." -> Recording stops. Local preview.
  • Farmer clicks "Upload to IPFS." -> SHA256 hash. Video to Pinata. IPFS CID returned. Data to localStorage.

Functional Req:

  • REQ-3.2.1: Allow video recording (WebRTC).
  • REQ-3.2.2: Input video metadata (crop, location, pesticide, company, purpose).
  • REQ-3.2.3: Calculate SHA256 of video pre-upload.
  • REQ-3.2.4: Upload video/metadata (inc. hash) to IPFS (Pinata API).
  • REQ-3.2.5: Store IPFS CID & metadata (inc. hash) in farmer's localStorage.
  • REQ-3.2.6: Feedback on upload status.

3.3 Farmer: AgriStream Video Management

4. External Interface Requirements

4.1 User Interfaces

Responsive web GUI (Vue.js, Tailwind CSS). Views: Landing, Login/Onboarding, Farmer/Buyer/Investor Dashboards, Profile, Messaging. Adheres to web design principles; modals for focused tasks; mobile-friendly menu.

4.2 Hardware Interfaces

User devices (desktops, laptops, tablets, smartphones) with modern browsers. Camera for video. Standard inputs.

4.3 Software Interfaces

Modern browsers. OS (Win, macOS, Linux, Android, iOS). APIs: Orange ID (Bedrock SDK), Solana Wallets (window.solana), Pinata (HTTP POST), Google Gemini (HTTP POST), Groq (HTTP POST). Browser localStorage.

4.4 Communications Interfaces

HTTP/HTTPS for web traffic/APIs. IPFS Gateway (HTTPS) for video retrieval.

5. Other Nonfunctional Requirements

5.1 Performance Requirements

Page load: 3-5s. Video upload: user/API dependent; feedback needed. API responses: Gemini 10-20s, Groq 2-5s. UI responsiveness: <200ms. localStorage: efficient.

5.2 Safety Requirements

Demo: no real financial/sensitive data beyond basic profile. SHA256 for (simulated) video integrity. Gemini API safety settings. AI advice informational.

5.3 Security Requirements

Auth: Orange ID, Solana wallet security. API Keys: Client-side (DEMO ONLY), production needs backend. IPFS: Public by CID. localStorage: Not app-encrypted. HTTPS for production. XSS: Vue.js protection; care with v-html.

5.4 Software Quality Attributes

  • Usability: Intuitive, clear feedback, mobile responsive.
  • Reliability (Demo): Consistent, graceful error handling, localStorage persistence.
  • Maintainability (Demo): Structured Vue code, some comments.
  • Portability: Web app.
  • Testability (Demo): Manual.
  • Modularity (Conceptual): Distinct features.

6. Other Requirements

API Key Config: User/dev MUST use valid Pinata, Gemini, Groq keys. Demo Data: Loads initial data to localStorage if empty. Browser Permissions: User grants camera access, etc.

Appendix A: Glossary

PestiVid: A prototype agricultural platform using blockchain and AI to verify crop quality through video submissions.

Web3: The decentralized web powered by blockchain, enabling dApps (decentralized applications) with user-owned data.

IPFS: InterPlanetary File System — a peer-to-peer hypermedia protocol designed to make the web faster, safer, and more open.

Pinata: A service that allows you to manage and pin files to IPFS to ensure persistent availability across the network.

CID: Content Identifier — a unique hash used in IPFS to locate and verify the integrity of a file.

SHA256: A cryptographic hash function used to generate secure and unique data fingerprints.

Solana: A high-performance blockchain supporting fast and low-cost decentralized apps (simulated in this project).

Orange ID: An authentication service for managing user identities securely within the PestiVid ecosystem.

Gemini API: A Google AI model integrated for identifying plant diseases and recommending pesticide solutions.

Groq API: An ultra-fast LLM API used to power the AgriBot chatbot for farmer interaction and support.

AgriSell: A module where farmers can list and sell their verified crops in a transparent marketplace.

AgriStream: A feature that hosts verified farmer-submitted crop videos, ensuring visual transparency.

PlantRecommend: An AI-powered module that analyzes plant health and offers recovery suggestions based on visual inputs.

AgriBot: An intelligent chatbot that provides real-time assistance to farmers using LLMs and domain-specific training.

localStorage: A web browser feature used as a temporary demo database to store user information on the client side.

SRS: Software Requirements Specification — this document outlines the goals, functions, and structure of the PestiVid system.

GUI: Graphical User Interface — the visual component of the system users interact with to access platform features.

API: Application Programming Interface — a set of defined rules that enable different software modules to communicate.

JWT: JSON Web Token — a compact and self-contained method for securely transmitting authentication data between parties.

ROI: Return on Investment — a performance measure used to evaluate the efficiency or profitability of an investment.

SOL: The native cryptocurrency of the Solana blockchain used in the simulated transaction flows.

WebRTC: A web technology enabling real-time communication of audio, video, and data directly between browsers.
